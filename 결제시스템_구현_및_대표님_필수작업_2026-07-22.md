# XENA 결제 시스템 구현 기록

## 현재 선택

Stripe Checkout을 1차 결제 PG로 연결한다. 현재 코드와 기존 안내 문서가 Stripe Checkout을 전제로 하고 있어 가장 적은 변경으로 국제 카드 결제를 시작할 수 있다. 국내 간편결제와 계좌이체가 필요해지는 시점에는 Toss Payments 또는 다른 국내 PG를 별도 검토한다.

## 구현된 서버 흐름

1. 로그인한 사용자가 상점에서 `안전하게 구매`를 누른다.
2. Firebase Callable Function `createCheckoutSession`이 서버에 고정된 상품 ID와 금액을 확인한다.
3. 서버가 Stripe Checkout Session을 만들고 사용자를 Stripe 결제 화면으로 보낸다.
4. Stripe가 `stripeWebhook`으로 결제 결과를 보낸다.
5. 웹훅 서명을 검증하고 `purchases/{stripeSessionId}`를 멱등 처리한다.
6. 결제가 확인된 경우에만 `wallets/{uid}.shards`를 서버에서 증가시킨다.
7. Stripe 환불 이벤트가 오면 해당 구매를 `refunded`로 만들고 지급된 파편을 회수한다. 이미 사용한 파편은 `premiumShardDebt`로 기록한다.

## 상품 금액

| 상품 ID | 지급 | 가격 |
|---|---:|---:|
| AS_120 | 변칙 파편 120 | 1,500원 |
| AS_650 | 변칙 파편 700 | 7,500원 |
| AS_1400 | 변칙 파편 1,600 | 15,000원 |

금액은 클라이언트가 아니라 `functions/index.js`의 서버 상품표를 기준으로 한다.

## 대표님이 해야 하는 작업

### 1. Stripe 계정 생성 및 사업자 심사

Stripe Dashboard에서 사업자·정산 계좌·고객지원 정보를 등록한다. 처음에는 테스트 모드로 진행하고, 테스트 결제가 확인된 뒤 라이브 모드로 전환한다.

### 2. Firebase Secret Manager에 키 등록

PowerShell에서 공식 웹사이트 Firebase 프로젝트 폴더로 이동한 뒤 실행한다.

```powershell
firebase functions:secrets:set STRIPE_SECRET_KEY --project xena-c012f
firebase functions:secrets:set STRIPE_WEBHOOK_SECRET --project xena-c012f
```

키 입력 시 화면에 값을 남기거나 채팅창에 붙여넣지 않는다. `STRIPE_SECRET_KEY`는 Stripe의 테스트 또는 라이브 Secret key이고, `STRIPE_WEBHOOK_SECRET`은 아래 웹훅 엔드포인트를 등록한 뒤 Stripe가 제공하는 `whsec_...` 값이다.

### 3. Stripe 웹훅 등록

Stripe Dashboard의 Developers → Webhooks에서 다음 주소를 등록한다.

```text
https://us-central1-xena-c012f.cloudfunctions.net/stripeWebhook
```

이벤트는 최소한 다음 세 가지를 선택한다.

- `checkout.session.completed`
- `checkout.session.async_payment_succeeded`
- `charge.refunded`

생성된 Signing secret을 `STRIPE_WEBHOOK_SECRET`에 등록한다.

### 4. Functions 및 Firestore Rules 배포

```powershell
firebase deploy --only functions:createCheckoutSession,functions:stripeWebhook,firestore:rules --project xena-c012f
```

그 다음 테스트 카드로 결제하고 Firebase Console의 `purchases`, `wallets` 문서를 확인한다. 결제 화면으로 돌아온 직후 웹훅 처리에 수 초가 걸릴 수 있다.

## 아직 라이브 전 확인할 것

- Stripe 테스트 결제 1회와 환불 1회
- 같은 Checkout Session 웹훅을 반복 전송해도 파편이 한 번만 지급되는지
- 로그인하지 않은 사용자가 결제 버튼을 눌렀을 때 차단되는지
- 환불 시 파편 잔액과 `premiumShardDebt`가 기록되는지
- 개인정보처리방침에 Stripe 결제 처리와 결제 식별자 보관을 추가했는지
- 실제 사업자명, 환불 문의 채널, 미성년자 결제 안내를 법적 문서에 입력했는지

## 비용 메모

Firebase Functions와 Secret Manager는 사용량에 따라 과금될 수 있고 Stripe는 결제 수수료가 발생한다. 현재 구현에는 별도 서버를 추가하지 않았으므로 운영 고정비를 늘리지 않는 방향이다. 실제 요율과 세금·환불 규정은 Stripe와 사업자 등록 상태에 따라 달라질 수 있으므로 라이브 전 Dashboard에서 확인한다.

## 주의

현재 변칙 파편 결제 지급은 서버 장부로 보호되지만, 기존 코스메틱 구매 중 일부는 아직 브라우저 저장 상태를 함께 사용한다. 정식 유료 운영 전에는 코스메틱 보유 목록도 `purchaseCosmetic` 서버 함수와 계정 컬렉션으로 이전해야 한다.
